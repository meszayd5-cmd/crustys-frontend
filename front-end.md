# Crusty’s Express – Frontend Specification (Phase 1)

## 📌 Project Scope

This document defines the complete Frontend structure and vision for the official Crusty’s Express website.

⚠️ Important:
This phase includes **Frontend development only**.
Backend logic, payment integration, and delivery APIs are not included in this phase.

---

# 1. Project Objective

Build a modern, premium, urban-style website reflecting the strong visual identity of Crusty’s Express.

The website must:

- Represent the brand professionally
- Showcase all available delivery products
- Allow adding products to a cart (frontend logic)
- Prepare for future delivery integration
- Deliver a smooth and responsive experience

---

# 2. UX/UI Vision

The website must feel:

- Urban
- Bold
- Premium
- Organized
- Modern
- Fast

We will keep the restaurant's existing identity:

- Black dominant
- Red primary
- Yellow accent
- Neon / Urban vibe

No rebranding.
No color changes.

---

# 3. Website Structure

---

## 3.1 Hero Section (Animated Entry)

Purpose:
Create a short and impactful first impression.

Characteristics:

- Dark immersive background
- Logo with subtle neon glow effect
- Smooth fade-in animation
- Short headline
- "Explore Menu" button
- Animation duration: 1.5–2 seconds maximum

⚠️ No heavy 3D.
⚠️ No long cinematic intro.
⚠️ Must remain fast and smooth.

---

## 3.2 Navigation Bar

Top navigation must include:

- Logo
- Menu
- About
- Contact
- Cart Icon

Behavior:

- Sticky on scroll
- Minimal and clean
- Dark theme

---

# 4. Menu Structure

All delivery products must be included.

Categories:

- Beef Burgers
- Chicken Burgers
- Baguette Sandwiches
- Wraps
- Loaded Fries
- Platters
- Chicken Tenders
- Appetizers
- Cold Drinks
- Extras

Each category must be clearly separated and easy to navigate.

---

# 5. Product Card Structure

Each product card must contain:

- Product Image (HD, clean background)
- Product Name
- Short Description
- Price
- Promotion Badge (if applicable)
- "Add to Cart" button

Design Requirements:

- Dark card background
- Clean layout
- Subtle hover animation
- Smooth appearance animation

---

# 6. Cart (Frontend Only)

Cart must allow:

- Add product
- Increase / decrease quantity
- Remove product
- Dynamic total calculation

After clicking "Checkout":

Display:

"Choose your delivery partner"

Options:

- Uber Eats
- DoorDash
- SkipTheDishes

⚠️ No API integration in this phase.
Only frontend UI logic.

---

# 7. Design System

## Colors

- Deep Black (primary background)
- Red (primary CTA)
- Yellow (accent highlights)
- White (text)

Glow effects must be subtle.
No visual overload.

---

## Typography

- Bold titles
- Clean readable body text
- Strong hierarchy

---

# 8. Animations

Allowed:

- Fade in
- Slide up
- Hover effects
- Smooth scroll

Not allowed:

- Heavy 3D scenes
- Slow loading animations
- Over-complicated effects

---

# 9. Responsive Requirements

Must work perfectly on:

- Mobile (priority)
- Tablet
- Desktop

Mobile-first approach required.

---

# 10. Performance

- Optimized images
- Lazy loading
- Clean code structure
- Fast loading time (<3 seconds)

---

# 11. Final Expected Result

The website must look like:

✅ Official brand website  
✅ Premium fast-food brand  
✅ Modern and organized  
✅ Not a copy of Uber Eats  
✅ Not overloaded  
✅ Not basic  

This is a strong brand presentation website with ordering preparation capability.
